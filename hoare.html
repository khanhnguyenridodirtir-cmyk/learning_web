Cẩm nang Chứng minh Chương trình

Hoare Logic từ lý thuyết cơ bản đến thực chiến phân tích thuật toán.

1. Bộ ba Hoare (Hoare Triple) là gì?

Mọi chứng minh chương trình đều xoay quanh công thức cơ bản này:

{ P } C { Q }
  • { P } - Tiền điều kiện (Precondition): Trạng thái bắt buộc của hệ thống trước khi chạy lệnh.
  • C - Lệnh (Command): Đoạn code sẽ được thực thi.
  • { Q } - Hậu điều kiện (Postcondition): Trạng thái của hệ thống chắc chắn đạt được sau khi chạy xong lệnh.

2. Các Luật Chứng minh Cốt lõi

Đây là các "công thức toán" để bạn thao tác với code. Hãy đặc biệt chú ý đến Luật Gán và tư duy làm ngược.

A. Luật Gán (Phép Thế - Substitution)

Bí quyết lớn nhất: LÀM NGƯỢC TỪ DƯỚI LÊN. Ta ĐÃ BIẾT Hậu điều kiện muốn đạt được. Ta tìm Tiền điều kiện bằng cách thay thế biến bằng giá trị gán.

{ Q[E/x] } ← Thay mọi chữ x trong Q bằng E
x = E;
{ Q } ← Hậu điều kiện muốn đạt tới

B. Luật Hệ Quả (Implied)

Dùng để "nới lỏng" hoặc "thắt chặt" các điều kiện toán học mà không cần chạy qua dòng code nào. Nếu P mạnh hơn P', ta có thể thay P bằng P'.

Nếu P &implies; P' và { P' } C { Q }
Thì suy ra: { P } C { Q }

C. Luật Nối tiếp (Sequence)

Dùng để nối nhiều dòng code lại với nhau. Điều kiện đầu ra của lệnh 1 chính là điều kiện đầu vào của lệnh 2.

{ P }
C1;
{ R } ← Trạng thái trung gian
C2;
{ Q }

D. Luật Vòng lặp (While) cơ bản

Yêu cầu phải có một Bất Biến (Loop Invariant - I). Đây là một sự thật luôn luôn ĐÚNG ở 3 thời điểm: trước, trong mỗi chu kỳ, và sau khi thoát vòng lặp.

{ I }
while (B) {
  { I ∧ B }
  C;
  { I }
}
{ I ∧ ¬B } ← Thoát vòng lặp

3. Phân tích sâu: Chứng minh cấu trúc Rẽ nhánh (If - Else)

Khi gặp lệnh if (B) { C1 } else { C2 }, chương trình sẽ chia làm 2 vũ trụ song song. Để chứng minh Hậu điều kiện { Q } là đúng, ta phải chứng minh cả 2 nhánh đều phải dẫn đến cùng một cái đích { Q }.

Công thức suy diễn logic (Formal Rule):
{ P ∧ B } C1 { Q } (Nhánh IF)
{ P ∧ ¬B } C2 { Q } (Nhánh ELSE)
{ P } if (B) C1 else C2 { Q }
BÍ KÍP THỰC CHIẾN

Cách giải ngược (Bottom-Up) cho If-Else:

Trong thực tế, ta thường biết trước { Q } (cái đích cần đến), và biết trước { P } (tiền điều kiện đầu vào). Các bước làm chuẩn xác là:

  • Dùng Luật Gán đẩy { Q } ngược lên qua lệnh C1 để tìm ra điều kiện Q1.
  • Dùng Luật Gán đẩy { Q } ngược lên qua lệnh C2 để tìm ra điều kiện Q2.
  • Dùng Luật Hệ quả (Implied) chứng minh 2 điều toán học sau:
    1. ( P ∧ B ) &implies; Q1
    2. ( P ∧ ¬B ) &implies; Q2
💡 Lưu ý: Tại sao lại có bước 3? Vì khi đi vào nhánh IF, ta chắc chắn rằng biểu thức điều kiện B là đúng. Nên ta được quyền gộp P và B lại để làm điểm tựa suy luận ra Q1. Tương tự với nhánh ELSE, ta có điểm tựa là P và ¬B.

4. Phân tích sâu: Vòng lặp While & Bí ẩn của "Bất Biến" (Invariant)

Khác với lệnh gán hay if-else, vòng lặp while có thể chạy số lần không xác định. Để chứng minh, ta không thể chạy tay từng vòng. Ta phải tìm ra một Bất biến (Loop Invariant - Ký hiệu là I).

Bất biến (Invariant) thực chất là gì?

Hãy tưởng tượng Bất biến là một "Chiếc mỏ neo". Dù vòng lặp có cuộn trào bao nhiêu lần, chiếc mỏ neo này luôn luôn giữ một sự thật toán học không bao giờ thay đổi.
Nó phải đủ yếu để đúng ngay từ khi chưa chạy vòng lặp nào, nhưng phải đủ mạnh để khi kết hợp với điều kiện thoát vòng lặp, nó giúp ta suy ra được cái đích cuối cùng.

3 Bài kiểm tra khắt khe của Bất Biến (I):

1

Khởi tạo (Init)

Trước khi vào loop, I phải đúng.

Tiền đ.kiện &implies; { I }
2

Bảo toàn (Maintain)

Chạy 1 vòng xong, I vẫn phải đúng.

{ I ∧ B } Code { I }
3

Kết thúc (Terminate)

Thoát loop, I giúp suy ra kết quả.

{ I ∧ ¬B } &implies; { Q }

Làm sao để tự nghĩ ra Bất Biến (Invariant)?

Tìm Bất biến là phần khó nhất. 90% trường hợp, Bất biến I chính là Hậu điều kiện { Q } bị "làm yếu đi". Dưới đây là 2 kỹ thuật phổ biến nhất:

Thủ thuật 1: Thay hằng số bằng biến

Biến Hậu điều kiện từ "hoàn thành toàn bộ" thành "hoàn thành một phần".

Mục tiêu (Q): S = Tổng(1..N)
Biến N thành biến đếm i.
=> Bất biến (I): S = Tổng(1..i)
Thủ thuật 2: Xóa bớt một điều kiện

Nếu Q có dạng A ∧ B, ta lấy A làm Bất biến, và ¬B làm điều kiện vòng lặp.

Mục tiêu (Q): Mảng sắp xếp ∧ i == N
Lấy nửa đầu làm Bất biến.
=> Bất biến (I): Mảng đã sắp xếp đến i.

THỰC CHIẾN CHỨNG MINH LINE-BY-LINE

Áp dụng form trình bày đầy đủ các bước (Assignment, Implied, Invariant) cho mọi dòng code.

Ví dụ 1: Tính Giai thừa (Bài mẫu từ hình ảnh)

Đây là bài toán mẫu chuẩn xác nhất thể hiện sức mạnh của Luật Gán lùi (Assignment) kết hợp với Bảo toàn Bất biến (Invariant).

... Khởi tạo ban đầu ...
(| y = z! |) Pre-condition
while ( z != x ) {
(| y = z! ∧ z ≠ x |) Invariant
(| y · (z + 1) = (z + 1)! |) Implied
z = z + 1;
(| y · z = z! |) Assignment
y = y * z;
(| y = z! |) Assignment
}
(| y = z! ∧ ¬(z ≠ x) |) Partial-while
(| y = x! |) Implied
Giải thích Logic (Đọc từ dưới lên):
1. Để có y = z! sau lệnh y = y * z, ta thay y bằng y * z → Thu được y * z = z!
2. Để có y * z = z! sau lệnh z = z + 1, ta thay z bằng z + 1 → Thu được y * (z + 1) = (z + 1)!
3. Bất biến y = z! hiển nhiên thỏa mãn tính chất toán học để suy ra biểu thức ở bước 2. Chứng minh thành công!

Ví dụ 2: Tính tổng từ 1 đến N

{ N ≥ 0 } Pre-condition
{ 0 = Tổng(1..0) ∧ 0 ≤ N } Implied
i = 0;
{ 0 = Tổng(1..i) ∧ i ≤ N } Assignment
S = 0;
{ S = Tổng(1..i) ∧ i ≤ N } Assignment (Invariant)
while ( i < N ) {
{ S = Tổng(1..i) ∧ i ≤ N ∧ i < N } Invariant
{ S + (i + 1) = Tổng(1..i+1) ∧ i + 1 ≤ N } Implied
i = i + 1;
{ S + i = Tổng(1..i) ∧ i ≤ N } Assignment
S = S + i;
{ S = Tổng(1..i) ∧ i ≤ N } Assignment
}
{ S = Tổng(1..i) ∧ i ≤ N ∧ ¬(i < N) } Partial-while
{ S = Tổng(1..N) } Implied

Ví dụ 3: Tìm LCA (Tổ tiên chung)

Viết lại thuật toán LCA của bạn thành dạng line-by-line để thấy rõ logic thế biến mảng.

while ( i < size_a ∧ i < size_b ∧ list_a[i] == list_b[i] ) {
{ lowest == list_a[i-1] ∧ (∀k < i, list_a[k] == list_b[k]) ∧ list_a[i] == list_b[i] } Invariant
{ list_a[i] == list_a[i] ∧ (∀k ≤ i, list_a[k] == list_b[k]) } Implied
lowest = list_a[i];
{ lowest == list_a[i] ∧ (∀k ≤ i, list_a[k] == list_b[k]) } Assignment
i = i + 1;
{ lowest == list_a[i-1] ∧ (∀k < i, list_a[k] == list_b[k]) } Assignment
}
{ I ∧ ¬(Điều kiện lặp) } Partial-while
{ lowest == LCA } Implied
Ở dòng Implied bên trong vòng lặp, ta đã khéo léo biến đổi (∀k < i) ∧ (tại i) thành (∀k ≤ i). Đồng thời chèn thêm mệnh đề luôn đúng list_a[i] == list_a[i] để dọn đường cho lệnh gán lowest = list_a[i] ngay bên dưới.

Ví dụ 4: Thuật toán Trộn (Merge Array) - Rẽ nhánh

Áp dụng chuẩn line-by-line cho cấu trúc IF-ELSE nằm trọn trong lòng Bất biến vòng lặp.

{ I: Mảng [start..run-1] đã sắp xếp ∧ (run == start + i + j) } Invariant đầu loop
if ( left_array[i] < right_array[j] ) {
{ I ∧ left_array[i] < right_array[j] } If-Condition
{ I[left_array[i] / *run, run+1 / run, i+1 / i] } Implied
*run = left_array[i];
{ I[run+1 / run, i+1 / i] } Assignment
i = i + 1;
{ I[run+1 / run] } Assignment
} else {
{ I ∧ left_array[i] ≥ right_array[j] } Else-Condition
{ I[right_array[j] / *run, run+1 / run, j+1 / j] } Implied
*run = right_array[j];
{ I[run+1 / run, j+1 / j] } Assignment
j = j + 1;
{ I[run+1 / run] } Assignment
}
{ I[run+1 / run] } ← Cả 2 nhánh đều hội tụ về đây Rule of If-Else
run = run + 1;
{ I } Assignment
Ký hiệu: I[A / B] nghĩa là lấy biểu thức I, thay mọi chữ B bằng A. Rõ ràng cả 2 vũ trụ rẽ nhánh cuối cùng đều chuẩn bị sẵn sàng một trạng thái giống hệt nhau để phục vụ cho lệnh run = run + 1 phía dưới.
