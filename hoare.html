Cẩm nang Chứng minh Chương trình

Hoare Logic từ lý thuyết cơ bản đến thực chiến phân tích thuật toán.

1. Bộ ba Hoare (Hoare Triple) là gì?

Mọi chứng minh chương trình đều xoay quanh công thức cơ bản này:

{ P } C { Q }
  • { P } - Tiền điều kiện (Precondition): Trạng thái bắt buộc của hệ thống trước khi chạy lệnh.
  • C - Lệnh (Command): Đoạn code sẽ được thực thi.
  • { Q } - Hậu điều kiện (Postcondition): Trạng thái của hệ thống chắc chắn đạt được sau khi chạy xong lệnh.

2. Các Luật Chứng minh Cốt lõi

Đây là các "công thức toán" để bạn thao tác với code. Hãy đặc biệt chú ý đến Luật Gán và tư duy làm ngược.

A. Luật Gán (Phép Thế - Substitution)

Bí quyết lớn nhất: LÀM NGƯỢC TỪ DƯỚI LÊN. Ta ĐÃ BIẾT Hậu điều kiện muốn đạt được. Ta tìm Tiền điều kiện bằng cách thay thế biến bằng giá trị gán.

{ Q[E/x] } ← Thay mọi chữ x trong Q bằng E
x = E;
{ Q } ← Hậu điều kiện muốn đạt tới
Ví dụ (Tại sao `{list_a[i] == list_a[i]}`?):
Ta muốn sau khi gán, lowest == list_a[i].
Lệnh là: lowest = list_a[i];
=> Thay chữ lowest bằng list_a[i] vào điều kiện dưới.
Kết quả (Tiền điều kiện): { list_a[i] == list_a[i] } (Luôn ĐÚNG).

B. Luật Hệ Quả (Implied)

Dùng để "nới lỏng" hoặc "thắt chặt" các điều kiện toán học mà không cần chạy qua dòng code nào. Nếu P mạnh hơn P', ta có thể thay P bằng P'.

Nếu P &implies; P' và { P' } C { Q }
Thì suy ra: { P } C { Q }
Ý nghĩa: Khi bạn thấy chữ implied, nghĩa là: "Từ khẳng định A ở trên, theo logic toán học hiển nhiên suy ra được khẳng định B ở dưới, ta kết nối chúng lại."

C. Luật Nối tiếp (Sequence)

Dùng để nối nhiều dòng code lại với nhau. Điều kiện đầu ra của lệnh 1 chính là điều kiện đầu vào của lệnh 2.

{ P }
C1;
{ R } ← Trạng thái trung gian
C2;
{ Q }

D. Luật Vòng lặp (While)

Yêu cầu phải có một Bất Biến (Loop Invariant - I). Đây là một sự thật luôn luôn ĐÚNG ở 3 thời điểm: trước, trong mỗi chu kỳ, và sau khi thoát vòng lặp.

{ I }
while (B) {
  { I ∧ B }
  C;
  { I }
}
{ I ∧ ¬B } ← Thoát vòng lặp

3. Giao diện Khối Chứng minh

Để dễ đọc, chúng ta sẽ gộp các bước chứng minh thành các "Khối" (Block). Mỗi khối hoàn chỉnh một Bộ ba Hoare:

{ Tiền điều kiện (Precondition): Trạng thái trước khi chạy }
// Code thực thi
// Các bước suy luận toán học (Implied/Luật hệ quả) nằm ở đây
{ Hậu điều kiện (Postcondition): Trạng thái sau khi chạy }

Ví dụ 1: Tính tổng từ 1 đến N

Mục tiêu (Hậu điều kiện cuối cùng):

{ S = Tổng các số từ 1 đến N }

Bước 1: Khởi tạo

{ N ≥ 0 }
int i = 0;
int S = 0;
{ I: S = Tổng(1..i) ∧ i ≤ N }

Bước 2: Vòng lặp (Bảo toàn Bất biến)

{ I ∧ (i < N) }
Luật Gán lùi: Muốn có { S = Tổng(1..i) } sau lệnh i++, trước đó cần { S = Tổng(1..i-1) }.
Muốn có { S = Tổng(1..i-1) } sau lệnh S = S + i, trước đó cần { S + i = Tổng(1..i) }.
S = S + i;
i++;
{ I }

Bước 3: Thoát vòng lặp

{ I ∧ ¬(i < N) }
Implied: (i ≤ N) ∧ (i ≥ N) &implies; i = N.
Thay i = N vào I: S = Tổng(1..N)
{ S = Tổng(1..N) }

Ví dụ 2: Tìm Tổ tiên chung gần nhất (LCA)

Đây là bài toán của bạn. Chúng ta sẽ dùng Luật Gán lùi và Luật Hệ Quả (Implied) để giải thích chính xác tại sao các lệnh lại móc nối được với nhau.

Mục tiêu cuối cùng:

{ lowest == LCA (Tổ tiên chung gần nhất) }

Bất biến I: lowest == list_a[i-1] ∧ (Mọi k < i, list_a[k] == list_b[k])

Bước 1: Thiết lập cấu trúc

{ list_a[0] == list_b[0] == root }
int i = 1;
int lowest = list_a[0];
{ I: lowest == list_a[i-1] ∧ (Mọi k < i, list_a[k] == list_b[k]) }

Bước 2: Giải phẫu nội tạng Vòng lặp (Mấu chốt vấn đề)

Làm sao từ đầu vòng lặp, chạy qua 2 lệnh gán, ta lại quay về đúng Bất Biến ban đầu? Hãy đọc ngược từ dưới lên.

// ĐẦU VÒNG LẶP (Có Bất biến VÀ Điều kiện Loop)
{ I ∧ (list_a[i] == list_b[i]) }
Luật Hệ Quả (Implied): Từ khẳng định phía trên, ta biết mọi node trước `i` giống nhau (nhờ I) VÀ node tại `i` cũng giống nhau (nhờ ĐK Loop).
=> Suy ra (Implies): Mọi node từ `0` đến `i` là giống nhau. Ta gọi trạng thái này là P2.
{ P2: Mọi k ≤ i, list_a[k] == list_b[k] }

lowest = list_a[i];

Luật Gán (Tính ngược): Dựa vào trạng thái P1 ở dưới, thay `lowest` bằng `list_a[i]`, ta được list_a[i] == list_a[i] ∧ (Mọi k ≤ i, list_a[k]...).
Vì list_a[i] == list_a[i] luôn đúng, nó bị triệt tiêu. Phần còn lại khớp y hệt P2 ở trên!
{ P1: lowest == list_a[i] ∧ (Mọi k ≤ i, list_a[k] == list_b[k]) }

i++;

Luật Gán (Tính ngược): Lấy Hậu điều kiện { I } ở tuốt dưới cùng, thay mọi chữ i thành (i+1).
lowest == list_a[(i+1)-1] => Trở thành lowest == list_a[i]
Mọi k < (i+1) => Trở thành Mọi k ≤ i
=> Ghép lại, nó khớp chính xác 100% với trạng thái P1 ở trên!
// CUỐI VÒNG LẶP (Đã bảo toàn thành công Bất biến)
{ I: lowest == list_a[i-1] ∧ (Mọi k < i, list_a[k] == list_b[k]) }

Bước 3: Thoát vòng lặp & Kết luận

{ I ∧ (i ≥ size_a ∨ i ≥ size_b ∨ list_a[i] != list_b[i]) }
Implied: Do Bất biến I bảo đảm mọi phần tử trước i giống nhau, nên khi vòng lặp dừng lại (nghĩa là 2 nhánh bắt đầu rẽ hướng hoặc đã chạm đích), phần tử chung cuối cùng chính là (i-1). Mà lowest hiện đang lưu list_a[i-1]. Suy ra lowest là LCA.
{ lowest == LCA }

Trang web này bao hàm đầy đủ từ Khái niệm Hoare Triple, Luật Gán, Luật Hệ Quả cho tới phân tích Block.
